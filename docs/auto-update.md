# Auto-update

BlogWriter checks GitHub Releases for new versions and can install them
in-place. Updates are **ed25519-signed**: the app ships a public key and only
ever installs artifacts whose signature verifies — a compromised download
server cannot push a malicious binary.

## How it works (user side)

- On startup the app silently checks
  `https://github.com/kwhorne/BlogWriter/releases/latest/download/latest.json`.
- If a newer version exists, an **update x.y.z** button appears in the toolbar
  (and under **Settings → Updates**).
- **Install & restart** downloads the artifact, verifies its signature,
  swaps the installed app, and relaunches:
  - **macOS**: replaces the running `BlogWriter.app` bundle (rolls back on failure)
  - **Linux**: replaces the `blogwriter` executable and re-execs
- You can also check manually via **Settings → Check for updates**.

Auto-update only works from an installed build (`.app` / installed binary),
not from `cargo run`.

## How it works (release side)

Components:

| Piece | Where |
|---|---|
| Private signing key | `~/.config/blogwriter/update_key` (base64 ed25519 seed) — **never in git** |
| Public key | Embedded in `src/updater.rs` (`PUBLIC_KEY_B64`) |
| Signing tool | `cargo run --bin blogwriter-sign` — `keygen` / `pubkey` / `sign` |
| Manifest | `dist/latest.json`, generated by `scripts/make-update-manifest.sh` |

Release flow:

```bash
scripts/build-macos.sh            # signed + notarized .app, .dmg, .zip
scripts/build-linux.sh            # linux tar.gz
scripts/make-update-manifest.sh   # sign artifacts -> dist/latest.json
gh release create vX.Y.Z dist/BlogWriter-* dist/latest.json --title ... --notes ...
```

`latest.json` must be attached to the release — the app resolves it through
the stable `releases/latest/download/latest.json` URL.

### Manifest format

```json
{
  "version": "1.1.0",
  "notes": "See the release page",
  "platforms": {
    "macos-aarch64":  { "url": "https://github.com/.../BlogWriter-1.1.0-macos-arm64.zip", "signature": "<base64>" },
    "linux-aarch64":  { "url": "https://github.com/.../BlogWriter-1.1.0-linux-arm64.tar.gz", "signature": "<base64>" }
  }
}
```

The platform key is `{os}-{arch}` (`std::env::consts`). The macOS artifact is
the **zip of the .app** (not the DMG); Linux uses the tar.gz.

## Key rotation

If the private key leaks: generate a new pair (`blogwriter-sign keygen`),
update `PUBLIC_KEY_B64` in `src/updater.rs`, and ship one release signed so
that existing installs can still verify it — i.e. rotate across two releases,
or accept that older installs must update manually.
